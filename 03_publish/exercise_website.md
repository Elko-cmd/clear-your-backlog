Eine eigene Website ist das wichtigste Werkzeug der kuenstlerischen Selbstpraesentierung. Sie ist eure digitale Visitenkarte, euer Portfolio, euer Archiv -- und oft der erste Eindruck, den Kurator*innen, Galerist*innen oder Juror*innen von eurer Arbeit bekommen. Diese Uebung fuehrt durch Planung, Struktur und Umsetzung einer eigenen Kuenstler*innen-Website.

---

## Warum eine eigene Website?

- **Unabhaengigkeit** -- Social Media gehoert Plattformen, eure Website gehoert euch
- **Professionalitaet** -- eine eigene Domain signalisiert, dass ihr es ernst meint
- **Langlebigkeit** -- Instagram-Posts verschwinden im Feed, eine Website bleibt auffindbar
- **Kontrolle** -- ihr bestimmt Layout, Reihenfolge, Kontext und Praesentation
- **Bewerbungen** -- fast jede Ausschreibung fragt nach einer Website

---

## Anatomie einer Kuenstler*innen-Website

### Pflicht-Seiten

| Seite | Inhalt | Tipps |
|---|---|---|
| **Startseite** | Starkes Bild oder Auswahl, Name, evtl. kurzer Satz | Weniger ist mehr. Kein Text-Wall. |
| **Arbeiten / Works** | Portfolio eurer Arbeiten mit Bild, Titel, Jahr, Medium, Kurzbeschreibung | Qualitaet vor Quantitaet. 5-10 starke Arbeiten reichen. |
| **Ueber / About** | Kuenstlerisches Statement, Kurzbiografie | Nutzt euer Statement aus dem Seminar. |
| **CV** | Ausstellungen, Ausbildung, Stipendien, Preise | Tabellarisch und aktuell halten. |
| **Kontakt** | E-Mail (kein Kontaktformular noetig), ggf. Social Media Links | Eine professionelle E-Mail-Adresse verwenden. |

### Optional (aber empfohlen)

| Seite | Inhalt |
|---|---|
| **Aktuell / News** | Kommende Ausstellungen, Publikationen, Neuigkeiten |
| **Texte / Press** | Pressetexte, Katalogbeitraege, Interviews |
| **Impressum** | In Deutschland Pflicht bei oeffentlich zugaenglichen Websites |

## Werkseite: Wie praesentiere ich eine Arbeit?

Jede Arbeit sollte eine eigene Unterseite oder einen eigenen Abschnitt haben:

1. **Titel**, Jahr
2. **Medium** (z.B. "Videoinstallation, 3-Kanal, 12 Min.")
3. **2-5 gute Abbildungen** (Installationsansichten, Details, Stills)
4. **Kurzbeschreibung** (3-5 Saetze -- nutzt eure Texte aus Uebung 1)
5. **Optional:** Ausstellungskontext, Credits, Video-Embed, Tech Rider Link

**Bildqualitaet:**
- Professionelle Dokumentation ist entscheidend (siehe Uebung Dokumentation)
- Web-optimiert: 1500-2500px Breite, 72dpi, sRGB
- Dateigroesse unter 500KB pro Bild (schnelle Ladezeit!)
- Einheitlicher Stil bei Installationsansichten

## Technische Umsetzung

### Option 1: Website-Baukasten (kein Code noetig)

Siehe die ausfuehrliche Liste in [Baukästen.md](Website/Baukästen.md).

Empfehlungen fuer Kuenstler*innen:
- **[Cargo](https://cargo.site)** -- speziell fuer Kreative, kostenlos fuer Studierende
- **[Carrd](https://carrd.co)** -- minimalistisch, guenstig, fuer Onepager
- **[Squarespace](https://squarespace.com)** -- elegante Templates, einfach zu bedienen
- **[Framer](https://www.framer.com/)** -- interaktiv, fuer Designaffine
- **[newmediakassel.de](https://newmediakassel.de)** -- Klassenseite Neue Medien, Account bei Elko/Joel/Leonie

### Option 2: Statische Website (HTML/CSS)

Fuer diejenigen, die mehr Kontrolle wollen:
- Kostenlos hostbar bei [Neocities](https://neocities.org) oder [GitHub Pages](https://pages.github.com)
- Volle Gestaltungsfreiheit
- Lerneffekt: HTML/CSS sind nuetzliche Grundkenntnisse
- Beispiel-Template im Ordner [HTML-Übung/](HTML-Übung/)

### Option 3: Dynamische Website (JS/PHP)

Fuer technisch Fortgeschrittene:
- Nachlade-Websites mit JavaScript 
- Hosting bei [uberspace](https://uber.space) oder [wint.global](https://wint.global)

### Domain und Hosting

Eine eigene Domain ist wichtig:
- **Domain-Anbieter:** [Namecheap](https://namecheap.com), [wint.global](https://wint.global) (.de inklusive), [Strato](https://strato.de)
- **Hosting:** [uberspace](https://uber.space), [wint.global](https://wint.global), [Strato](https://strato.de)
- **Tipp:** `vorname-nachname.de` oder `vorname-nachname.com` sichern, solange verfuegbar
- Viele Baukästen bieten eigene Domains an (oft im Abo enthalten)

## Dos and Don'ts

### Do

- **Wenige, starke Arbeiten zeigen** -- lieber 5 gute als 30 mittlere
- **Aktuell halten** -- eine veraltete Website schadet mehr als keine
- **Mobilfreundlich** -- die meisten Besucher*innen kommen vom Handy
- **Schnelle Ladezeit** -- Bilder komprimieren, keine Auto-Play-Videos auf der Startseite
- **Einfache Navigation** -- Besucher*innen sollen in 2 Klicks jede Arbeit finden
- **Impressum** -- rechtlich verpflichtend in Deutschland

### Don't

- **Kein Flash, keine Musik, keine Ladebildschirme** -- es ist nicht 2005
- **Keine Stockfotos** -- nur eigene Arbeiten und Dokumentation
- **Nicht zu viel Text auf der Startseite** -- die Arbeiten sprechen zuerst
- **Keine toten Links** -- regelmaessig pruefen
- **Keine passwortgeschuetzten Bereiche** (ausser fuer spezifische Bewerbungen)
- **Nicht nur Instagram einbetten** -- das ist keine Website

## Uebung (ca. 45 Min)

### Phase 1: Analyse (10 Min)

Recherchiert Websites von Kuenstler*innen, die ihr gut findet. Schaut euch jeweils an:
- Wie ist die Seite strukturiert?
- Wie werden Arbeiten praesentiert (Raster, Slider, Einzelseiten)?
- Was macht die Seite professionell?
- Was wuerdet ihr anders machen?

Teilt eure Fundstücke in der Gruppe.

### Phase 2: Struktur planen (15 Min)

Plant eure eigene Website. Erstellt eine Sitemap (Stift und Papier reicht):

1. Welche Seiten brauche ich?
2. Welche Arbeiten zeige ich (Auswahl!)?
3. In welcher Reihenfolge?
4. Welche Texte habe ich schon (Statement, CV, Werkbeschreibungen)?
5. Welche Bilder habe ich, welche brauche ich noch?

### Phase 3: Umsetzung starten (15 Min)

Waehlt einen technischen Weg (Baukasten oder Code) und legt los:
- Registriert einen Account bei einem Baukasten **oder** legt eine HTML-Datei an
- Richtet die Grundstruktur ein (Navigation, Seiten)
- Ladet eine erste Arbeit hoch mit Bild und Text
- Nutzt eure vorhandenen Texte aus dem Obsidian Vault

### Phase 4: Peer Review (5 Min)

Zeigt euren aktuellen Stand. Feedback:
- Findet man sofort die Arbeiten?
- Ist die Navigation klar?
- Wirkt es professionell?
- Funktioniert es auf dem Handy?

## Website-Pflege im Obsidian Vault

Nutzt den Ordner `03_Website/` in eurem Vault:

1. **Sitemap** als Notiz anlegen (welche Seiten, welche Arbeiten)
2. **Texte vorbereiten** -- Statement, About, Werkbeschreibungen aus dem Vault uebernehmen
3. **Bilderliste** -- welche Bilder brauche ich noch, welche muessen nachbearbeitet werden?
4. **Checkliste vor Go-Live:**
   - [ ] Alle Texte Korrektur gelesen
   - [ ] Bilder komprimiert und einheitlich
   - [ ] Navigation auf Handy getestet
   - [ ] Impressum vorhanden
   - [ ] Kontaktdaten aktuell
   - [ ] Domain eingerichtet
   - [ ] Freund*in hat Probe-gesurft
